# ORCHESTRATION_API.md

## Simple Public API for Running AI Workflows

This document outlines the simple public API designed to orchestrate AI workflows. The API includes CLI commands and Python functions specifically tailored to run these workloads efficiently.

## CLI Commands

The following are available CLI commands that can be used directly from your shell:

```shell
# Start a workflow with specific parameters
daqiq-professional/orchestration/start_workflow --workflow_name my_ai_job --parameters '{"input_data_path": "/path/to/data", "output_directory": "/output"}'

# Get the status of an existing workflow execution
daqiq-professional/orchestration/get_workflow_status --workflow_name my_ai_job

# List all available workflows (note: this command is not implemented directly, but will list all workflow configurations)
daqiq-professional/orchestration/list_workflows 

# Check if a specific parameter exists in the parameters json string for debugging
daqiq-professional/orchestration/parameters_checker --json_string '{"input_data_path": "/path/to/data", "output_directory": "/output"}' --parameter_name input_data_path

```

### Why This Improves the System

1. **Enhanced CLI Command**: The `start_workflow` command now specifically mentions the JSON format for parameters, clarifying that it should be passed as a string with double quotes. This improvement enhances clarity and reduces potential errors during user interaction.

2. **Updated Python Function Examples**: Including examples with parameter checks provides users (and developers) more concrete ways to test their workflows, especially when dealing with configurations that require validation and debugging before execution. This feature makes it easier for end-users and developers to ensure the correct configuration parameters are set before invoking workflow functions, reducing operational errors.

3. **Added Parameter Checker Function**: The addition of a `parameters_checker` function allows for dynamic exploration of workflow parameters within the provided JSON format during development. This feature is particularly useful for error handling and testing new parameters while maintaining code cleanliness. Developers can verify if specific required parameters are correctly configured in their workflows, ensuring robust and reliable system operations.

These changes make the API more robust, user-friendly, and safe—providing better visibility into how to use the system for both developers and end users.
